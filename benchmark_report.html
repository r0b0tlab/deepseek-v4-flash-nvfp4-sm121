🔧 Workarounds Applied

1. SwiGLU Limit Patch

# Patched config.json on both nodes sed -i 's/"swiglu_limit": 10.0/"swiglu_limit": null/' config.json

Allows flashinfer_b12x backend to load (SM121-specific CUTLASS backend).

2. Eager Mode

# Disabled torch.compile and CUDAGraphs llm = LLM( ..., enforce_eager=True, )

Bypasses auto_functionalized compilation error.

3. Ray Cluster on Port 6380

# Node 1 (head) ray start --head --port=6380 --node-ip-address=192.168.100.10 # Node 2 (worker) ray start --address='192.168.100.10:6380' --node-ip-address=192.168.100.11

🛠️ Recommended Next Steps

PyTorch FP8 Support

Upgrade PyTorch to a build with SM12.1 FP8 ScalarType 44 support. Requires NVIDIA PyTorch 2.6+ with Blackwell support.

SwiGLU Kernel Update

Wait for vLLM to add SwiGLU clamp support to flashinfer_b12x backend, or for flashinfer_trtllm to support SM12.1.

Alternative: DeepSeek-V3

Switch to DeepSeek-V3 (236B params, ~37B active). No published NVFP4 checkpoint, but quantization pipeline is documented.

Alternative: Container Deployment

Use NVIDIA's NGC container with pre-built PyTorch + vLLM for Blackwell.
