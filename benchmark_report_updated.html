📋 Executive Summary

Critical Finding: The NVIDIA DeepSeek-V4-Flash-NVFP4 checkpoint uses NVFP4 quantization, but the previously successful DSV4-Flash deployment on SM12.1 used a MXFP4 checkpoint with a custom vLLM build. These are incompatible quantization formats.

After extensive investigation using the proven DSV4-Flash SM121 vLLM build (commit dda4668b), we discovered that:

  • The pinned build was designed for MXFP4 MoE, not NVFP4
  • All MXFP4 backends (deep_gemm, triton, flashinfer_trtllm) fail on SM12.1 with "kernel does not support current device cuda"
  • The NVFP4 path in vLLM 0.22.1 supports flashinfer_b12x but fails with PyTorch FP8 ScalarType 44

⚠️ Critical Blockers

DSV4-007: PyTorch FP8 ScalarType 44 (vLLM 0.22.1)
Not yet supported ScalarType 44 — PyTorch on GB10 lacks float8_e4m3fn support. The model requires FP8 KV cache which is mandatory for DeepSeek-V4.
DSV4-008: MXFP4 vs NVFP4 Quantization Mismatch (Pinned Build)
The proven SM12.1 vLLM build (dda4668b) was designed for MXFP4 MoE, but the NVIDIA checkpoint uses NVFP4. All MXFP4 backends fail on SM12.1 with "kernel does not support current device cuda".
DSV4-009: No Working Backend for NVFP4 on SM12.1
The flashinfer_b12x backend (NVFP4 SM12.1) works in vLLM 0.22.1 but is blocked by FP8 ScalarType 44. The pinned build has no flashinfer_b12x backend at all.

🎯 Root Cause Analysis

The DSV4-Flash deployment that achieved c=1 38.4 t/s on SM12.1 used a DIFFERENT checkpoint with MXFP4 quantization.

The NVIDIA DeepSeek-V4-Flash-NVFP4 checkpoint is a different quantization variant that requires:

  • NVFP4 MoE kernels (not MXFP4)
  • flashinfer_b12x backend for SM12.1 (only in vLLM 0.22.1+)
  • PyTorch with FP8 float8_e4m3fn ScalarType 44 support

The current PyTorch 2.11.0+cu130 build does support FP8 (hasattr(torch, "float8_e4m3fn") == True), but the vLLM 0.22.1 code path hits a different FP8 scalar type that is not yet supported in the stable value conversions.

🛣️ Path Forward

Combine the NVFP4 path from vLLM 0.22.1 with the PyTorch 2.11.0+cu130 and CUDA 13.2 environment from the pinned build. Requires significant build engineering.

Wait for PyTorch 2.6+ with full SM12.1 FP8 support and vLLM update with NVFP4 flashinfer_b12x backend. Timeline unknown.
